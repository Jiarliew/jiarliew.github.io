Dust

Dust is a private mobile messenging application. Send public post to followers, or send secure messages that can be deleted permanently on the other persons device.

Problem

The task was to completely redesign the existing application. Instead of searching for design problems internally, feedback from current users gave insight to problems we wouldn't be able to find.

Step One: Discovery

Survey

Surveys were sent out to current users to ask what they liked and disliked about the app. After reading the responses, results showed that the UI was a glaring problem. It was confusing all around the app, from the navigation down to the color.

Initial User Testing

Viewing users interact with the app provided insight to additional problems they faced when using the app, from onboarding to changing settings.

Pain points discovered from user test and surveying.

Major pain points included a confusing navigation, unclear terminology, ambiguous icons, and lack of hierarchy. Because of all this, users had a hard time navigating the app in general. Which ultimately led to users being frustrated and leaving the app.

Prioritizing

When creating a list of major pain points, many small problems were also discovered from user testing. Prioritizing pain points helped the design team focus on the bigger issues when designing, while keeping the smaller problems in mind.

Prioritizing pain points from high to low.

Step Two: Discovery

Navigation

The first major change was to create a centralized navigation. Users were confused with the existing app that had two bottom buttons and three top tabs. We tested various design iterations to find the one with the least pain points according to the users.

User testing different navigation styles.

A bottom navigation deemed most effective. Creating a five tab navigation, with the important features, seemed to let users use the app with less confusion.

Naming Convention

Users were still struggling to understand what certain tabs did because of the branded terminology and the ambiguous icons that were used. This led to users having to tap around the navigation a couple times to fully understand. Creating more conventional text and icons was the next problem to solve.

Users had a hard time understanding where tabs lead to with the current naming conventions

The main tabs that were confusing to users were the "dust" and "blast" screen. The terminology was marketed with the app name in mind, but made no sense to new users. We questioned users with various tab names to see what users would make of it. It was quickly discovered that the simpler terms were easier to undertsand.

Color

Multiple colors made the app look inconsistent, and also confused users at time for what was clickable. Using one color would help place emphasis on the important items that would help guide the user. The apps main feature was security, and users needed to trust using the app. Blue was the perfect color. After testing the various colors, almost all users associated the app to Facebook instantly.

The color blue conveys trust.

We didn't want new users to quickly associate our app with another, we decided to not use blue. This led to trying out other colors. Orange was another color that conveyed security, which was something the app needs users to believe.

Because blue was too associated with Facebook, orange was another color that conveyed a sense of security.

Outcome

After knocking out the larger problems, there was still a lot of work to be done. Smaller issues still played a large part in creating a great app. Efficiently using whitepace and font weight helped to create hierarchy. Creating informative empty screens helped to guide users around the app. Simplifying the onboarding process generated more accounts created.

Before and after.

After many iterations and testing all over the app, the redesign was complete. There were many changes to the app, from onboarding, to icons, to colors, and even completely changing the name and logo. Without user testing, we never could've spotted or corrected many of the existing problems. Utilizing user test was a fantastic way to help improve our app.
