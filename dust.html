Dust

Dust is an mobile messenging application that sends secure messages that can be deleted permanently.

Problem

The task was to redesign the existing app. The first step was to receive feedback of the app from current users to see how they feel about the app.

Step One: Discovery

Survey

Results from users showed that the the UI was a glaring problem. The UI was confusing from the navigation down to the color.

Initial User Testing

Viewing the users interact with the app provided better insight to the problems they faced.

Major pain points included a confusing navigation, unclear terminology, ambiguous icons, and lack of hierarchy.Users had a hard time finding and understanding the important features because of this. This ultimately led to users being frustrated and leaving the app.

Prioritizing

Along with the major pain points, there were many small problems that were discovered from user testing. Prioritizing would help focus on the bigger issues when designing, and also help keep in mind the smaller issues for the future.

Step Two: Discovery

Navigation

The first major change was to create a more centralized navigation. Users were confused with the existing app that had two bottom buttons and three top tabs that leaded to different screens. We tested various iterations to find the one with the least pain points according to the users.

A bottom navigation deemed most effective. Creating a five tab navigation, with the most used/important features for users. Combining the existing profile button into the navigation, and adding a contacts tab.

Naming Convention

Users were still struggling to understand what certain tabs did because of the branded terminology and the ambiguous icons that were used, that led to users having to tap around to fully understand. Creating more conventional text and icons was the next problem to solve.

The main tabs that were confusing to users were the 'dust' and 'blast' screen. The terminology was marketed with the app name in mind, but made no sense to new users. We questioned users with various tab names to see what users would make of it. It was quickly discovered that the simpler terms were easier to undertsand.

Color

Something had to be done with the existing color palette. Multiple colors placed all over the app created no sense of hierarchy.

To simplify the app, using one color would help place emphasis on the important items that would help guide the user. The apps main feature was security, and users needed to trust using the app. Blue was the perfect color. After testing the various colors, almost all users associated the app to Facebook instantly. This led to trying out other colors. Orange was a color that conveys security, which was something the app needs users to believe.

Home Stretch

After knocking down the larger problems, there was still work to be done. Smaller issues still plays a large part in creating a good app. Efficiently using whitepace and font weight helped to create hierarchy. Guiding users around the app with informative empty screens, and also simplifying the onboarding process. After many iterations and testing, the redesign was complete. User testing was a fantastic way to discover the problems users faced with the app.
